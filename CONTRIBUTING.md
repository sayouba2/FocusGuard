# Contributing to FocusGuard

FocusGuard is an early-stage open-source project. Contributions should improve the clarity, safety, or technical credibility of the prototype.

## Before you start

- Read the README and current roadmap.
- Open an issue before working on larger changes.
- Keep changes small and reviewable.
- Prefer local data and platform-supported Android APIs.

## Suggested workflow

1. Fork or branch from `main`.
2. Describe the problem in an issue.
3. Keep one change per commit when possible.
4. Add tests or a reproducible manual check if the change affects behavior.
5. Update the README or roadmap when you change project scope.

## What to avoid

- Hidden tracking or analytics.
- Blocking emergency access.
- Large unreviewed rewrites.
- Scope creep that turns the project into generic device-lock software.

## Coding style

- Use clear names and small widgets.
- Document any Android limitation directly in the code or README.
- Prefer simple Flutter and Dart features over premature abstraction.

## Reporting bugs

Please include:

- device model
- Android version
- Flutter version
- steps to reproduce
- expected result
- actual result
